# Report Template: Comparative Research

Use this structure when the user has specific options and needs to choose between them.
Give one default recommendation per dimension. Do not present equal-weight comparison tables — the user came to avoid making a technical judgment.

---

## Report structure

### Background
3 sentences: who is making this decision, in what context, and what core problem it solves.
Must be understandable by someone not in this conversation.

### Requirements & Constraints
Restate all requirements and constraints from user input and Phase 0 answers.

### Decision Dimensions

#### \<Dimension 1\>
**Recommended**: \<specific option\> — \<one-line reason tied to user's constraints\>
**Alternatives**: \<option A\> if \<specific condition\>; \<option B\> if \<specific condition\>

#### \<Dimension 2\>
...

### Common Pitfalls
- ...

### Recommendation
**Choose**: \<specific combination across all dimensions\>
**Why**: \<one sentence per dimension\>
**Avoid**: ...
**Defer**: ...

### Open Questions
Up to 2 implementation-level decisions that research alone cannot resolve.
Omit section entirely if nothing is genuinely unresolved.
Each question must be answerable in one sentence.
