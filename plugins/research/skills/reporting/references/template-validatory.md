# Report Template: Validatory Research

Use this structure when the user has a specific plan or approach and wants to know if it is correct.
The goal is a clear verdict: correct, needs modification, or replace entirely.

---

## Report structure

### Background
3 sentences: who has this plan, in what context, and what they are trying to validate. Must be understandable by someone not in this conversation.

### What Is Being Validated
Restate the user's proposed approach precisely, as you understand it. If the user's framing was imprecise, state the corrected version and explain why.

### Evidence For
Findings that support the user's approach:
- \<finding\> — \<source or reasoning\>
- ...

### Evidence Against
Findings that challenge or contradict the approach:
- \<finding\> — \<source or reasoning\>
- ...

If no contradictory evidence was found after thorough search, state this explicitly. Do not invent weaknesses.

### Verdict

**\<CORRECT / NEEDS MODIFICATION / REPLACE ENTIRELY\>**

**Why**: \<one paragraph explaining the verdict based on the evidence above\>

If NEEDS MODIFICATION:
**What to change**: \<specific modifications\>

If REPLACE ENTIRELY:
**Use instead**: \<specific alternative\>
**Why it fits better**: \<one sentence\>

### Common Pitfalls of This Approach
Failure modes that practitioners have hit with this method — even if the approach is generally correct.
- ...

### Open Questions
Up to 2 decisions that research cannot resolve — only the user's specific context can answer them.
Omit section entirely if none.
Each question must be answerable in one sentence.
