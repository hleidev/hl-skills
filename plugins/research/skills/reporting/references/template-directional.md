# Report Template: Directional Research

Use this structure when the user has a goal but no path. The user may have mentioned one approach —
but the value of this report is surfacing viable routes they did not know about, not elaborating on the one they guessed.

---

## Report structure

### Background
3 sentences: who has this goal, in what context, and what the core problem is. Must be understandable by someone not in this conversation.

### Goal & Constraints
- **Goal**: what the user is trying to achieve, stated precisely
- **Hard constraints**: what cannot change
- **Challenged assumptions** (if any): approaches the user mentioned that research shows are suboptimal or incorrect, and why

### Viable Paths

#### Path 1: \<name\>
**What this looks like**: concrete description of this approach
**Pros**: ...
**Cons / risks**: ...
**Best when**: specific conditions under which this path is the right choice

#### Path 2: \<name\>
...

Aim for 2–4 paths. Include at least one path the user likely had not considered. Do not invent implausible paths — only include approaches that real practitioners use.

### Recommended Path
**Choose**: \<path name\>
**Why**: \<one sentence per key reason, tied to user's constraints\>
**First steps**: \<3 or fewer concrete actions to get started\>

### Common Mistakes
- ...

### Open Questions
Up to 2 questions that research cannot resolve — only the user can answer them. Omit section entirely if none.
Each question must be answerable in one sentence.
If a challenged assumption from Phase 0 remains unresolved, surface it here.
